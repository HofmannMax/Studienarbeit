Prototypische Realisierung eines Wissensnetzes (Knowledge Graph) mit einer GraphDatenbank  
 
 
Ziel der Arbeit ist der beispielhafte Aufbau eines Wissensnetzes (Knowledge Graph) durch die Integration 
mehrerer oeffentlicher Datenquellen.   
Wissensnetze spielen eine wichtige Rolle in Unternehmen wie Google, Facebook, eBay, usw. Nicht nur soziale 
Netzwerke, sondern auch Produkte und viele andere Daten lassen sich damit verwalten. 
In dieser Arbeit soll beispielhaft ein Knowledge Graph fuer Konferenzen umgesetzt werden. 
Dazu werden die Daten von Konferenzen (wie Strata) mit Daten aus sozialen Netzwerken angereichert.   
Die Arbeit umfasst folgende Aufgaben:  

- Beschreibung von Grundlagen wie Graphstrukturen (z.B. Property Graph, RDF, und aehnliches), 
	Modellierung von Graphdaten, Knowledge Graph 
- Einarbeitung in eine Graph-Datenbank, vorrangig Neo4j 
- Webscraping von mindestens 2 Konferenzseiten, z.B. Strata (https://conferences.oreilly.com/strata) 
	oder DOAG (https://2019.doag.org/de/home/) zur Erfassung von Konferenzdaten ueber mehrerer Jahre 
- Speicherung der Daten in einer Graph-Datenbank 
- Integration weiterer Daten aus sozialen Medien (z.B. Tweets, die den Konferenz-Hashtag verwenden), 
	LinkedIn oder Facebook (1-2 verschiedene soziale Medien) 
- Einfache grafische Aufbereitung der Daten als Webseite oder alternativ gerne auch Anwendung von 
	KI Methoden (z.B. Identifikation von Doppelgaenger)  
- Prototypische Realisierung  
- Regelmaessiger, scrumartiger Austausch mit dem Betreuer, um ueber Fortschritt, Hindernisse 
	und naechste Schritte zu sprechen
